## 5. החלטות עיצוב פדגוגי-טכני שלמדנו

### 5.1 שמע
- **Web Speech API לא יציב בעברית** — לפעמים אומר שמות אותיות במקום צלילים, לפעמים לא עובד בכלל בלי קול עברי מותקן
- **הפתרון: Edge TTS — MP3 מוכנים** — `edge-tts` Python package, קול `he-IL-HilaNeural`, איכות נטענת
- **דפדפנים חוסמים אוטופליי** עד לקליק ראשון — חובה מסך "בוא/י נתחיל"
- שמע על טעות = מקריא את המילה שנבחרה (פדגוגי — לימוד דרך טעות)

### 5.2 תמונות vs SVG
- **SVG וניל ידני לא משתווה ל-DALL-E** — ניסינו, הוחלף
- **חלוקת עבודה:** ChatGPT לתמונות (רקעים, נוני, מילים), קוד לאינטראקציה
- **עקביות נוני בין סצינות:** DALL-E לא משכפל דמות 100% — נוני נראה אחר בכל סצינה
- **הפתרון:** רקעים **בלי נוני**, ונוני כשכבת PNG שקופה אחת, אותה תמונה בכל המסכים
- **הסרת רקע לבן:** Python PIL — `noni-hero-transparent.png`

### 5.3 לייאאוט
- **aspect-ratio + width/height 100% יחד = רגיש** — height ו-width מבטלים את aspect-ratio
- **נכון:** `width: min(100vw, calc(100dvh * 16/9))` + `aspect-ratio: 16/9`
- **נוני בפינה:** `position: fixed` — לא חלק מהגריד, לא ידחק
- **בעיות opacity באנימציה:** החלפת `animation` class מאפסת `forwards` state — חובה `opacity: 1` בכל מצב

### 5.4 דמות נוני
- **נוני אחד** עם 4 מצבים (idle, helping, hint, cheering) דרך CSS transforms + filters
- **אין PNG נפרד לכל תנוחה** — היה רעיון אבל לא נדרש לרמת ה-MVP
- אם תרצי תנוחות נפרדות בעתיד: צריך לחזור ל-ChatGPT עם character sheet של 6-8 תנוחות

## 6. רעיונות למשחקונים נוספים באי 3 — לחקור בשיחה החדשה

הפעילות הנוכחית היא tap-match. לפי `interactive-primitives.md` יש 6 envelopes:
1. **tap-match** ← בנוי
2. **drag-build**
3. **trace-write**
4. **shared-read**
5. **mini-task**
6. **memory-sort**

לפי `book-of-environments.md` שלב 3 כולל:
- "לחיצה על אות משמיעה שם אות וצליל"
- "מעקב עם האצבע על האות משאיר קו מים זוהר לרגע"
- "לחיצה ממושכת על אות גורמת לה 'לפרוח' מתוך האלמוג"
- "התאמה נכונה בין אות לצליל גורמת לאות לצוף מעט"

### רעיונות אפשריים למשחקונים
- **trace-write** — לעקוב עם האצבע על האות בכיוון הנכון. SVG path animation. קו מים זוהר.
- **memory-sort** — להתאים בין אות לבין 3 תמונות שמתחילות בה, גרירה
- **letter-bloom** — לחיצה ארוכה על אלמוג → האות "פורחת" החוצה
- **listen-trace** — שומע צליל ומצייר את האות
- **find-the-letter** — חיפוש האות במשפט/בסביבה
- **letter-vs-letter** — דיפרנציאלי בין 2 אותיות קרובות (ת/מ או ב/ק)
- **memory-pairs** — זיכרון של אות-לצליל

### שאלות לבירור
- האם רוצים מגוון של envelopes באותו אי, או רק 1-2?
- האם להתאים את הקושי לרמת התלמיד (BKT/Tier MTSS)?
- איך משלבים את מזכרת השליטה — אחרי כל אות או רק בסוף?
- האם המשחקונים יוצגו ברצף או בבחירת התלמיד?

### מקורות השראה
- `engine/audio/he/` — יש את כל המילים והאותיות בעברית מוקלטות
- `engine/content/items/island-03-letter-recognition.json` — בנק 50 פריטים (אפשר להרחיב ל-11 אותיות = 110 פריטים)
- `engine/diagnostic.html` ו-`engine/screener.html` — מסכי אבחון קיימים שיכולים להיות בסיס
