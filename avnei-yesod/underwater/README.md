## תיקוני המלצה שעדיין לא בוצעו ב-ChatGPT

המלצות שאוסיף לתיקון לפני בנייה. לא נכנסו לקבצים — הם נשמרו AS-IS מ-ChatGPT.

1. **שלב 7 — "מערת זרמי המילים"** → להחליף ל**"לגונת הזרמים"** או **"זרמי המילים השלושה"**. המילה "מערה" מנוגדת לאסטרטגיה (אסוציאציית חושך/פחד), גם כשמוסבר "פתוחה ומוארת".
2. **שלב 18 — "מערת הרגשות הרכה"** → להחליף ל**"בריכת הרגשות"** או **"מפרץ ההרגשה"**. אותה סיבה.
3. **שלב 18 — "פרצופים"** → להחליף ל**"פנים"** או **"הבעות פנים"**. "פרצוף" טעון שלילית בעברית ילדית.
4. **שלב 12 — "אוצר המילים בעל-פה"** → להחליף ל**"תיבת האוצר של נוני"**. שאר הסביבות בעלות שמות פיגורטיביים, רק כאן יש שם פדגוגי-טכני.

## החלטות ארכיטקטוניות

### סטק: HTML+CSS+JS וניל — לא React

ה-Build Brief של ChatGPT (`claude-code-brief.md` חלק ב' סעיף 3) מציע React+TypeScript. **אל תיישמי את ההמלצה הזו.** הוחלט במאי 2026 לבנות ב-HTML+CSS+JS וניל מהסיבות הבאות:

1. **תאימות לסטק הקיים** — `engine/`, `audio/`, `vocab-bank`, `diagnostic.html`, `screener.html`, `demo.html` כולם HTML/JS וניל. React יוצר שני סטקים שלא יושבים יחד.
2. **ביצועים על טאבלט בית-ספר** — HTML וניל יעיל יותר מ-React כבד על טאבלטים בינוניים. ילדי כיתה א'-ב' לא יקבלו iPad Pro.
3. **תיק טכני נמוך** — אין build step, אין node_modules, פתיחה ישירה בדפדפן. שינוי מהיר ב-CSS/JS = איטרציה מיידית.
4. **GitHub Pages מיידי** — `https://impact-os.app/avnei-yesod/` כבר עובד עם קבצי HTML.

**מה כן לאמץ מה-Brief**:
- כל הפלטה (CSS variables)
- כל האנימציות (CSS animations + עקרונות הקצב)
- מנגנון הפידבק המדורג (3 רמות טעות)
- מבנה ה-data model (אפשר ב-JS וניל עם JSON)
- כל הרכיבים הוויזואליים (כ-Web Components או class-based)

**איפה לבנות**: `avnei-yesod/underwater-app/` (תיקייה חדשה, ליד `underwater/` שמכיל את התיעוד).

### הימנעות מ-Lovable

נוסה ב-21.5.2026 ונדחה: האילוסטרציות שיוצר Lovable גנריות מדי (אייקוני lucide-react, דגי קליפ-ארט, תמנון 4-זרועות). לא רמת Lingumi/Khan Kids/Sago Mini הנדרשת. כל האילוסטרציות נבנות פנימית כ-SVG ב-Claude Code.

## הקשר לארכיטקטורה הקיימת של אבני יסוד

- `architecture-mvp.md` (מסמך-אם של אבני יסוד) קובע: 22 איים = שלבים פדגוגיים, מנוע BKT, דשבורד בוקר למורה. הספר הזה הוא **העטיפה החזותית** של אותה ארכיטקטורה — הוא לא מחליף, הוא לובש.
- `engine/` (המנוע הקיים) — מספק את ה-BKT, ה-vocab-bank, ה-TTS עברית, את הדשבורד למורה. הספר הזה מספק את ה-presentation layer.
- כל סביבה כאן מתאימה ל-island_id מהמערכת הקיימת.
- מטבע "פנינים" כאן = המקבילה של "אבני-קסם" ב-`gamification-spec.md`. שני השמות תקפים — יוכרע בפיילוט.
