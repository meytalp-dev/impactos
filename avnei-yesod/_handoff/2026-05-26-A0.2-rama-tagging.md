## מה נעשה

הוספת 2 שדות חדשים ל-schema של כל פריט תרגול במערכת:

| שדה | טיפוס | משמעות |
|---|---|---|
| `rama_task_alignment` | int 1-10 | לאיזו מ-10 משימות ראמ"ה הפריט תורם |
| `peima_target`        | int 1-3  | באיזו פעימה (1=ספט-אוק · 2=ינו-פבר · 3=מאי-יוני) |

## החלטה דקה ששווה לציין

`island-03-items.json` משלב **גם** זיהוי אות **וגם** מודעות פונמית (`skills_tested: [1, 2]` נשמר בקובץ).
בחרתי `rama_task_alignment: 2` (פונולוגית) כ-primary כי **התשובה הנמדדת היא של פונמה**, לא שם של אות. שם האות מופיע רק כ-cue.

אם מיטל חושבת אחרת — שינוי של שורה בסקריפט והרצה מחדש.

## איך משתמשים בשדות החדשים — דוגמאות

```javascript
import { RAMA_TASKS, filterByRamaTask, validateRamaTagging } from './js/shared/item-schema.js';

// 1. דשבורד מורה — להציג רק פריטים שתורמים למשימה 1 (קריאת שמות אותיות)
const task1Items = filterByRamaTask(allItems, 1);

// 2. בדיקת mastery משולש (A0.3) — האם הילדה עברה את רף ראמ"ה במשימה הזו?
const taskInfo = RAMA_TASKS[item.rama_task_alignment]; // { name_he, threshold, ... }

// 3. וולידציה לפני event-log
const { valid, errors } = validateRamaTagging(item);
if (!valid) console.warn('Item missing rama tagging:', errors);
```

## נקודות פתוחות לבדיקה של מיטל

1. **`island-03-items.json` → משימה 2 או משימה 1?** — שמתי 2 (פונולוגית) כי השאלה היא על צליל, לא על שם אות. אם זה לא נכון — שורה אחת בסקריפט.
2. **`island-03-trace-paths.json` → משימה 1 בלבד?** — לא דחפתי משימה 9 (הכתבה) כי trace ≠ writing-from-dictation. אם בכל זאת רוצים שייספר ל-משימה 9 — צריך להוסיף שדה משני (`rama_task_alignment_secondary`) או שיוך כפול.
3. **שדה משני (`rama_task_alignment_secondary`)?** — בספק ה-MVP. כרגע single int. אם הדשבורד יראה שאנחנו מאבדים נראות על "פריט שעובד על 2 משימות" — נוסיף.
