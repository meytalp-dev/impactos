# MOY — Middle of Year Diagnostic
## Spec רשמי · 28.5.2026 · שלב א' (לפני קוד)

> **מה זה המסמך:** spec שמוביל את בניית MOY-Lite — כלי עזר אינטראקטיבי שמעדכן BKT באבני יסוד בהתבסס על אותם מבנים שראמ"ה בודקת באמצע השנה. **לא תחליף למבדק ראמ"ה הרשמי** (שמחויב להיות 1-on-1 פרטני, עם פריטים ייעודיים שאסור לחזור עליהם בתרגול).
>
> **למה צריך:** ראמ"ה מבצעת 3 מבדקים שנתיים: BOY · MOY · EOY. ב-`engine/screener.html` קיים BOY. ב-MOY (ינואר-פברואר) ראמ"ה מודדת **משימות 3+4** — שלב חדש שלא נמדד ב-BOY. בלי MOY שלנו, אבני יסוד "עיוורת" לפעימה 2: ה-BKT לא מתחדש לפי משימות שראמ"ה בפועל בודקת.
>
> **תלויות שנסגרו:** A.1 ✓ · A.3 ✓ · A.4 ✓ · A0.2 (rama_task_alignment) ✓ · A.5 (cold-start) ✓ · F.21A ✓

## §1 — הקשר: איפה MOY-Lite יושב

| שאלה | תשובה |
|---|---|
| מי בעצם מבצעת את ה-MOY הרשמי? | **המורה** — 1-on-1 פרטני עם הילדה, עם PDF של ראמ"ה ו-`madrich-mivdak-kriah-grade1.txt`. אבני יסוד **לא מחליפה** את זה. |
| מה אבני יסוד עושה אז? | **MOY-Lite** — כלי עזר אינטראקטיבי שמדמה את משימות 3+4 עם פריטים שונים, מעדכן BKT, ומספק תמונת מצב למורה לפני המבדק הרשמי. |
| למי זה עוזר? | למורה — לקבל החלטה איזה תלמידה מוכנה ל-MOY הרשמי. ולתלמידה — לתרגל את סוג המשימות. |
| מתי רץ? | **חלון ינואר-פברואר 2026** (גמיש). המורה מתחילה כשיש לה זמן, פר ילדה לפי קצב ההתקדמות. |

**עקרון מנחה:** MOY-Lite **תוסף ל-Tier 1**, לא הערכה רשמית. הוא מספק signal ל-BKT ול-teacher-rama, ולא תחליף ל-screener.

## §3 — Architecture: MOY-Lite

### 3.1 — מיקום

- **קובץ חדש:** `engine/moy-screener.html` (מקביל ל-`engine/screener.html` הקיים של BOY)
- **API חדש:** `js/shared/assessments.js` — לוגיקת שמירה/קריאה
- **Item pool חדש:** `engine/moy-items.json` — פריטי MOY-Lite (שונים מתרגול!)

### 3.2 — Schema של `state.assessments`

```js
state.assessments = {
  boy: {
    [studentId]: {
      date: 1727049600000,         // timestamp
      task_1: { score: 22, threshold: 18, status: 'pass' },
      task_2: { score: 6, threshold: 5, status: 'pass' },
      overall: 'pass',
      method: 'screener-html'      // איך נמדד
    }
  },
  moy: {
    [studentId]: {
      date: 1738368000000,
      attempts: [
        {
          date: 1738368000000,
          task_3: { score: 7, threshold: 7, status: 'pass' },
          task_4: { score: 14, threshold: 16, status: 'fail' }
        },
        {
          date: 1741737600000,      // attempt 2 — אחרי 5-6 שבועות (ראמ"ה דרישה)
          task_3: { score: 9, threshold: 7, status: 'pass' },
          task_4: { score: 17, threshold: 16, status: 'pass' }
        }
      ],
      latest_status: 'pass',
      next_review_due: null         // יוגדר אם status=fail
    }
  },
  eoy: { /* same structure */ }
};
```

### 3.3 — Key חדש ב-localStorage

`underwater-app:assessments` (נפרד מ-`underwater-app:v1` שבו `events`).

## §4 — Item Pool — פריטים שונים מתרגול!

**⚠️ חשוב פדגוגית:** ראמ"ה אוסרת להשתמש בפריטי המבדק כתרגול. אבני יסוד תרחיב את הכלל הזה: **פריטי MOY-Lite חייבים להיות שונים מפריטי תרגול רגיל**, אחרת ה-BKT יזוהם.

### 4.1 — משימה 3 (הבנת טקסט מושמע)

ראמ"ה: 7+/10 שאלות הבנה על טקסט קצר מושמע.

**MOY-Lite version:**
- 6 טקסטים קצרים (60-90 מילים), מוקלטים ב-AvriNeural
- 10 שאלות הבנה (multiple choice, 3 אופציות)
- הילדה שומעת → רואה 3 אפשרויות עם תמונות → בוחרת
- threshold: **7/10**

**תיוג JSON:**
```json
{
  "task_type": "moy_task_3",
  "passage_id": "moy3-001",
  "audio_url": "assets/audio/moy/passage-001.mp3",
  "questions": [
    {
      "q_id": "moy3-001-q1",
      "text": "מה השם של הילד בסיפור?",
      "options": [
        { "id": "a", "text": "דני", "image": "boy-1.svg" },
        { "id": "b", "text": "יוסי", "image": "boy-2.svg" },
        { "id": "c", "text": "אבי", "image": "boy-3.svg" }
      ],
      "correct": "b"
    }
    // ... 9 more
  ]
}
```

### 4.2 — משימה 4 (מודעות לשונית)

ראמ"ה: 16+/22 שאלות על מבנה שפה.

**MOY-Lite version:**
- 22 שאלות במגוון תחומים: סדר מילים · יחסי משמעות · מבנה משפט · החלפת מילה
- audio + multiple choice
- threshold: **16/22**

**דוגמת פריט:**
```json
{
  "task_type": "moy_task_4",
  "q_id": "moy4-001",
  "instruction_audio": "moy4-001-instruction.mp3",
  "instruction_text": "איזה משפט נכון?",
  "options": [
    { "id": "a", "text": "הילד אכל את העוגה" },
    { "id": "b", "text": "אכל הילד את העוגה" },
    { "id": "c", "text": "העוגה אכל הילד" }
  ],
  "correct": "a",
  "skill_tag": "word_order"
}
```

### 4.3 — תוכן: מי כותב?

**מיטל כותבת ידנית.** ה-spec הזה מגדיר את ה-structure. **תוכן 60 פריטים MOY-Lite + 6 קטעי שמע = משימה פדגוגית של מיטל**, לא של סוכן הקוד. סוכן הקוד יבנה את ה-infrastructure + 2-3 פריטים dummy לבדיקה.

## §5 — UI Flow

### 5.1 — כניסה ל-MOY-Lite

המורה (לא הילדה) מפעילה את MOY-Lite מתוך **teacher-rama → Student View**:

```
┌──────────────────────────────────────────────────────────┐
│ Section 6 — הערכת אמצע שנה (MOY)                          │
│                                                            │
│ סטטוס: 🟡 לא נמדד עדיין                                   │
│ מומלץ: ינואר-פברואר 2026                                   │
│                                                            │
│ ⚠️ הערה: MOY-Lite הוא תרגול, לא תחליף למבדק ראמ"ה הרשמי. │
│   המבדק הרשמי 1-on-1 דורש PDF + מדריך מורה.              │
│                                                            │
│ [📋 הפעילי MOY-Lite] [📖 פתחי PDF ראמ"ה]                  │
└──────────────────────────────────────────────────────────┘
```

לחיצה → `engine/moy-screener.html?student=<id>&task=3` (התלמידה לבד עם הוראות מוקלטות).

### 5.2 — מסך התלמידה

```
┌──────────────────────────────────────────────────────────┐
│ [רקע ים רגוע]                                              │
│                                                            │
│       🔊 [נוני]: "הקשיבי טוב לסיפור הקצר..."              │
│                                                            │
│       [▶ נגן את הסיפור]                                   │
│                                                            │
│       ──── אחרי ההקשבה ────                              │
│                                                            │
│       "מה השם של הילד בסיפור?"                            │
│       [תמונה: דני]  [תמונה: יוסי]  [תמונה: אבי]           │
└──────────────────────────────────────────────────────────┘
```

הילדה בוחרת. אין feedback מיידי על "נכון/לא נכון" — זה evaluation, לא תרגול.

### 5.3 — סיום

הילדה רואה: "מצוין, סיימת! חזרי לכיתה."
**אסור** להציג את הציון לתלמידה.

המורה מקבלת:
- ניוטיפיקציה ב-teacher-rama
- שורה ב-Section 6: סטטוס מעודכן (pass/near/fail)
- אם fail → הצעת B.7 + תזכורת לחזור בעוד 5-6 שבועות

## §6 — חוקי repeat (אם נכשלה)

לפי PDF ראמ"ה שורה 247:

1. **אם status=fail במשימה X:**
   - שמור `next_review_due = date + 5*7*24*60*60*1000` (5 שבועות)
   - הצג ב-teacher-rama Section 6: "🔴 נדרשת העברה חוזרת בתאריך YYYY-MM-DD"
   - הצע B.7 רלוונטי (אם spec של B.7 מוכן)

2. **כשמגיע התאריך:**
   - notification ב-teacher-rama: "⏰ הגיע מועד העברה חוזרת של MOY-Lite לנועה"
   - אם המורה ביצעה repeat — עדכן `attempts[]` array, חשב `latest_status` מחדש

3. **אם attempt 2 גם fail:**
   - הצג: "⚠️ שתי העברות מתחת לרף. שקלי הפנייה למומחה שילוב."
   - **לא** repeat שלישי אוטומטי — המורה מחליטה.
