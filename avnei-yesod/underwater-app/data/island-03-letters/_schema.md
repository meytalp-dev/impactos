## סכמת ה-JSON

```jsonc
{
  // ===== זיהוי =====
  "letter": "ש",                       // האות (תו עברי בודד)
  "letter_key": "shin",                // המפתח האנגלי (חייב להתאים לשם sound-X.mp3)
  "quest_id": "shin-tap",              // מזהה ייחודי לרישום ב-island3-quests:completed

  // ===== מכניקה =====
  "mechanic": "tap-all",               // אחד מ: "tap-all" | "pick" | "quest"
  "mechanic_config": {
    "totalTiles": 12,                  // tap-all בלבד
    "numTargets": 5,                   // tap-all בלבד
    "podsPerRound": 4                  // pick בלבד
    // quest אינו צריך mechanic_config (משתמש בברירת מחדל 5 שלבים)
  },

  // ===== דיסטרקטורים =====
  "distractors": ["ל", "ת", "מ", "ר", "ב", "ק", "נ"],

  // ===== מונה (top-bar) =====
  "counter": {
    "unit": "שמשות",                   // טקסט ב-"0/5 X"
    "total": 5                         // כמה יחידות לפני סיום
  },

  // ===== כותרת + פתיחה =====
  "title": "מסע השמש",
  "intro_html": "<p>נוני מחפש את השמש בים.</p><p>הקישו על כל ה-<strong>ש׳</strong>-ים בים — וכל אחת שתדליקו תחזיר עוד אור!</p>",

  // ===== אודיו =====
  // ⚠️ קריטי פדגוגית: ילדים בני 6 לא קוראים. ההוראה חייבת להיות מושמעת
  // קולית. אם intro_audio_keys ריק → הרמקול מנגן רק את צליל האות (לא מספיק).
  // אם in_game_prompt_audio_key חסר → המשחק מנגן רק sound-<letter-key>.mp3
  // בלי הקשר. שני השדות = חובה כדי שהמשחקון פועל מקצה לקצה.
  "intro_audio_keys": [                // AvriNeural — overlay רמקול: 2 קבצים ברצף (פתיחה + משימה)
    "intro-shin-quest",                // "מסע השמש! נוני מחפש את השמש בים."
    "intro-shin-mission"               // "בואו תקישו על כל אלמוגי האות שׁ׳ בים..."
  ],
  "in_game_prompt_audio_key": "find-shin",        // AvriNeural — הוראה במשחק: "מצאו את כל אלמוגי האות שׁ׳ בים." (אחרי mount + ב-auto-hint)
  "finale_audio_key": "finale-found-treasure",    // אופציונלי, ברירת מחדל נגזרת אם חסר

  // ===== סיום =====
  "completion": {
    "title": "מצאתם את כל השמשות!",
    "message": "אור-שמש זוהר הצטרף לעולם של נוני.",
    "souvenir": "שמש זוהרת"           // נשמר ב-localStorage עם addSouvenir()
  },

  // ===== Hero אופציונלי =====
  "hero": {
    "png": null,                       // נתיב יחסי לתמונת hero, או null
    "alt": ""                          // טקסט חלופי
  }
}
```

## מינימום נדרש כדי שמשחקון יעלה (smoke test)

המפתחות הבאים חייבים להיות מוגדרים:

- `letter` (תו עברי)
- `quest_id` (string ייחודי)
- `mechanic` (אחד מ-3)
- `counter.unit` + `counter.total`
- `completion.title`

## מינימום נדרש כדי שמשחקון יהיה **שמיש פדגוגית** (לילד שלא קורא)

מעבר ל-smoke test, חובה גם:

- `intro_audio_keys` — לפחות 2 קבצי AvriNeural (פתיחה נרטיבית + משימה מפורשת)
- `in_game_prompt_audio_key` — קובץ AvriNeural יחיד שמשמיע את ההוראה במשחק

בלי שניהם, ילד בן 6 שעוד לא קורא לא יוכל לדעת מה לעשות. הטמפלייט יעלה
כי הטכניקה תקינה, אבל החוויה תיכשל.

יצירת הקבצים: ראו `scripts/generate-shin-demo-audio.py` כדוגמה (3 שורות
טקסט עברי מנוקד → 3 MP3 דרך edge-tts + AvriNeural). D.15 צריכה ליצור
17×3=51 קבצים כאלה.

## 🎯 נושא נרטיבי — Full C: 4 קבוצות נושאיות (אישרה מיטל 27.5.2026)

D.15 בונה את 17 האותיות החסרות **ב-4 קבוצות נושאיות**. כל קבוצה חולקת:
- ויזואל ייעודי (SVG inline בצורת בועה / כוכב / צדף / דג)
- 2 קבצי אודיו משותפים: `intro-<theme>-quest` + `intro-<theme>-mission`
- counter unit + title + completion souvenir פר נושא

מה שמשתנה פר אות:
- `letter` + `letter_key` + `distractors` + `quest_id`
- קובץ אודיו אחד פר אות: `find-<letter-key>` ("מצאו את כל ה-X עם האות Y")

### שדה `theme` ב-JSON

כל קובץ פר אות חייב לכלול:
```jsonc
"theme": "bubbles" | "stars" | "shells" | "fish"
```

הערך נטען על-ידי `game-shell.js`, שמעביר אותו ל-`mechanic-tap-all.js`. ה-mechanic מוסיף class `theme-<theme>` ל-`.tap-field` כדי שה-CSS יחיל את הויזואל הנכון.

### למה הסדר הזה לבנייה ב-D.15

1. **כוכבי הים** (קבוצה ראשונה אחרי בועות) — לאמת שהתשתית theme-aware עובדת. מיטל מאשרת ויזואל.
2. **צדפי החוף** — שכפול של המתודה.
3. **להקת הדגים** — צורה מורכבת יותר (דג), טוב לדחות אחרי שיש ביטחון.

## איזה mechanic לבחור פר אות?

זו **החלטה פדגוגית של מיטל** — לא נקבע אוטומטית. ההמלצות הראשוניות מבוססות על
4 הז'אנרים שאומתו ב-23.5.2026 בפרפלקסיטי (משחקון פר אות = 10-15 חשיפות, אוטומציה
מגיעה מ-21 האיים האחרים):

| Mechanic | מתאים ל | למה |
|---|---|---|
| `tap-all` | אותיות עם שכיחות גבוהה במילים פשוטות (א, ל, נ, ש, י) | תרגול זיהוי מהיר בשדה רועש |
| `pick` | אותיות עם דיסטרקטורים חזותיים קרובים (ז↔ו, ר↔ד, ה↔ח) | נדרשת הבחנה דווקא |
| `quest` | אותיות שדורשות גם זיהוי-צליל וגם זיהוי-צורה (אותיות מורכבות) | תרגול עמוק יותר, חוויה ארוכה יותר |

⚠️ **חובה לאשר עם מיטל לפני בניית 17 קבצי דאטה ב-D.15** — זו לא החלטה אוטומטית
של סוכן הקוד.

## מה לא נכלל בסכמה (כוונה)

- **תיוגי ראמ"ה (`rama_task_alignment`, `peima_target`)** — נשארים ברמת ה-item
  ב-JSONs הקיימים (`island-03-items.json` וכו'), לא בקובץ פר-אות.
- **BKT state** — מגיע מ-`avnei-bkt-v1` ב-localStorage, לא מקובץ דאטה.
- **EPA dimensions** — מגיע מ-`avnei-epa-v1` ב-localStorage.
- **Pack assignment** — מגיע מ-`curriculum/packs/grade1-tashpaz/*.json` (פאזה C).
