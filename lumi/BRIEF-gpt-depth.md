# בריף להתייעצות עם GPT — לומי: העמקה פדגוגית (שלב C→E)

> העתיקי לצ'אט GPT חדש. נכתב כך שיהיה מובן בלי הקשר קודם.

---

## רקע קצר
**לומי (Lumi)** = אפליקציה ללימוד **אנגלית כשפה שנייה** לילדים דוברי-עברית, גיל רך, שלב מתחילים (Pre-A1→A1), פיילוט בבתי-ספר. **מדוברת קודם, קריאה קטנה בהמשך** — מיושר עם שלב ה-Pre-Foundation של משרד החינוך (שמיעה-דיבור לפני קריאה).

## מה כבר בנוי ועובד (חשוב — אל תציע לבנות מה שקיים)
- **מנוע-מסע שזור** שמנגן לכל מילה מיני-קשת: `meet → recognize → discriminate → comprehend → produce → function`, עם **מתכנן חזרה-מרווחת חוצה-סשנים** ו**פאנל-ראיות להורה** (בלי אחוזים).
- **BKT** עם שער-שליטה רב-הקשרי (מילה נחשבת "נשלטת" רק אחרי הצלחה ב-≥2 ייצוגים, ≥2 הקשרים, וגם הבחנה בין-דומים אמיתית).
- כל פריט-תוכן הוא JSON בסכמה קבועה (למטה). המנוע קורא את שדה `dimension`.
- **החלטות שכבר קיבלנו** (אל תערער): הפקה = מסלול-אומץ בלי ציון-הגייה; אין שיחת-AI פתוחה; `no_read:true` על הכל (טרום-אוריינות מכוונת).

## סכמת פריט (כדי שתחזיר תוכן מוכן-להדבקה)
```json
{
 "id": "food.snacksmeal.function.bread.01",
 "topic": "food", "subtopic": "snacksmeal", "no_read": true,
 "dimension": "function", "target": ["bread"], "measured": false,
 "prompt": { "audio_en": "Can I have some bread, please?", "gesture": "lumi rubs its tummy" },
 "task_audio": "Give Lumi the food.", "task_key": "give_it",
 "options": [ {"img":"bread","correct":true}, {"img":"egg","correct":false}, {"img":"milk","correct":false} ],
 "help_he": "לוּמִי רָעֵב וּמְבַקֵּשׁ לֶחֶם"
}
```
`comprehend` דומה אבל `measured:true, kc_comprehension:true` ו-`instruction_key:"find_it"`.

## מגבלת-מנגנון חשובה
`function` היום הוא **חד-פנייתי**: לומי אומרת משפט + משימה + הילד בוחר תמונה. זה **לא דיאלוג-תורות** אמיתי.

---

## הפערים בעומק שאנחנו צריכים בהם עזרה (זו הבקשה)

### 1. דיאלוג-תורות אמיתי ("דיברתי")
איך לעצב **מיקרו-דיאלוג של 2–3 תורות** לילד Pre-A1, שנשאר פשוט, לא-חוסם, ומבוסס-בחירה/כפתורים (בלי חובת מיקרופון)?
- מה **המנגנון המינימלי** (איך תור עובד, איך הילד "מגיב")?
- מה **סכמת-התוכן** לדיאלוג רב-תור (הרחבה של הסכמה למעלה)?
- 3 תסריטי-דיאלוג לדוגמה (ברכות, אוכל, רגשות) — סגורים וצפויים.

### 2. עומק-הבנה ("הבנתי")
ה-`comprehend` הנוכחי הוא מסגרת-אחת חוזרת ("Where is my ___", "I want ___"). איך **לדרג הבנה** בלי לצאת מגיל-הרך והתמונה?
- הדרגה: מילה-בהקשר → תכונה+שם ("the blue ball") → הוראה אחת → **שתי הוראות ברצף** → מיקרו-סיפור.
- דוגמאות-תוכן קונקרטיות ל-2 עולמות (אוכל/משפחה).

### 3. "נזכרתי" — שליפה בלי אפשרויות
היום החזרה-המרווחת בעצם **בודקת-זיהוי-מחדש**, לא שליפה אמיתית. איך לממש **שליפה בלי אפשרויות** לילד רך בלי חרדה?
- מדרגות-תמיכה (השהיה → צליל-ראשון → מודל-חלקי → אז אפשרויות).
- איך זה יושב לצד ה-BKT (סוג-ראיה נפרד).

### 4. פוניקס MVP ("קראתי") — שלב E
תוכנית **קטנה ומדורגת** של אות→צליל→מילה, **רק על מילים שכבר בלקסיקון השמיעתי** (cat/dog/red/sun/hat).
- רצף 4 מיקרו-משחקים, ואילו אותיות/מילים ראשונות.

### 5. איזה עולם להעמיק ראשון, ובאיזה סדר
נתון כיסוי-המימדים הנוכחי (X=קיים): כל העולמות יש meet/recognize/discriminate/produce; `comprehend` יש ב-animals/numbers/food/face/body/family; `function` יש רק ב-animals (+אוכל שהוספנו). איזה **2–3 עולמות** להעמיק ראשונים ולמה?

## פורמט תשובה מבוקש
1. מנגנון-דיאלוג + סכמה + 3 תסריטים.
2. סולם-הבנה מדורג + דוגמאות ל-2 עולמות.
3. מנגנון-שליפה (recall) + מדרגות.
4. פוניקס MVP (רצף + מילים ראשונות).
5. סדר-העמקה מומלץ (2–3 עולמות) + נימוק.
6. **אזהרת over-engineering:** איפה אנחנו מסבכים מדי לגיל הזה.
