# Spec · משחקון-סיכום חמישייה ("מסיבת הצירופים") — אבני יסוד

**תאריך:** 3.7.2026
**סטטוס:** תכנון מאושר · ממתין להכרעת נרטיב לפני בנייה
**מטרה:** משחקון חזרה מצטבר (cumulative review) לסוף פרק — מתרגל בבת-אחת חמישה אותיות שנלמדו ("חמישייה"), ועונה: "האם החמישייה יושבת?"

---

## למה זה קיים (הרציונל הפדגוגי)
עד היום כל אי מתרגל **יחידה אחת** (אות / תנועה / צירוף). אין רגע של **cumulative review** — עצירה שבה הילדה "אורזת" מקבץ שנלמד לאורך פרק זמן. משחקון-הסיכום ממלא את החסר, על שלושה עקרונות:
1. **Interleaving** — הסבבים מעורבבים בין 5 האותיות (לא ×5 על אותה אות) → זיהוי-בהקשר אוטומטי.
2. **Transfer דו-כיווני** — גם זיהוי (שומעת→בוחרת צירוף) וגם קריאה (רואה צירוף→בוחרת צליל).
3. **Differentiation מובנית** — יותר סבבים לצירופים החלשים פר-ילדה (`getWeakestCVs`).

## איך הסט נגזר (טכני)
עבור 5 אותיות החמישייה (מהתוכנית השנתית), הסט = כל מפתחות `per_cv` בצורת `<letter>_<group>` שקיימים לאותן אותיות.
דוגמה (חמישייה ת·מ·ר·ב·ק): `ת_a, ת_i, מ_a, מ_i, ר_a, ב_a, ב_shwa, ק_a` → 8 צירופים לחזרה.
- שקלול סבבים: `bkt.getWeakestCVs(sid, n, {letter})` — צירופים חלשים מקבלים יותר סבבים.
- fallback (אין דאטה): התנועות הקנוניות פר אות מהתוכנית / pilot pack.

## לולאת המשחק
1. **פתיחה** — נוני + סיפור-מסגרת (נרטיב לבחירה) → "בואו נראה מה זוכרים!"
2. **סבב (ערבוב):**
   - סוג A · **זיהוי**: נוני משמיע `cv-<letter>-<vowel>.mp3` → הילדה נוגעת בצירוף הנכון (בין כמה על הלוח).
   - סוג B · **קריאה**: הלוח מראה צירוף מנוקד → הילדה בוחרת את הצליל הנכון. **two-tap (תיקון פדגוגי, מיטל 3.7.2026):** הרמקולים נראים זהים → נגיעה ראשונה = האזנה חופשית (לא נמדדת), נגיעה שנייה על אותו רמקול = בחירה (רק היא נשלחת ל-BKT). אחרי טעות — סימון ההאזנה מתנקה (חייבת להקשיב שוב). בלי זה טעות-ניחוש-מיקום נרשמת כאי-ידיעה = false negative שמזהם את per_cv וגורר תרגול-יתר.
   - phoneme-group-accept: מַ/מָ = נכון (תואם המנוע).
3. **חגיגה** — כל נכון → אלמנט חגיגה (SVG פנינה/צדף) מצטרף. פס התקדמות.
4. **סיום** — "החמישייה יושבת! 🌟" (כל הצירופים ב-mastery) או "כמעט! עוד מבט על: בִּי, קֶ".
5. כל סבב שולח event פר-CV (`primary_island_id:4, target_letter, target_phoneme_group, is_correct`) → מזין את ה-BKT → סוגר את הטריגר האוטומטי.

## 🟡 ההכרעה החסרה: הנרטיב
לפי ה-checklist — **חובה סיפור אמיתי (בעיה→פעולה→פתרון), לא wrapper גנרי.** שלוש הצעות:
1. **מסיבת הצירופים / שרשרת הפנינים** — הצירופים שאספנו לאורך הפרק מתאספים לחגיגה; כל צירוף נכון = פנינה בשרשרת; סיום = שרשרת שלמה.
2. **אוצר הצדפים** — צדף גדול נפתח רק כשכל הצירופים של החמישייה חוזרים; כל נכון = צדף קטן נפתח; סיום = אוצר.
3. **הופעת השונית** — כל הדגים שלמדנו איתם חוזרים לבמה להופעה משותפת; כל צירוף = דג עולה לבמה; סיום = "כל השונית מופיעה יחד!".
