# מערכת אדמין — אבני יסוד + פולס

דף ניהול פנימי אחד שמרכז את כל הדאטה של שני המוצרים. עובד גם ב-`file://` (בלי שרת).

## הפעלה
פותחים את `admin/index.html` בדפדפן.

## רענון הדאטה
כל הדאטה מחולצת אוטומטית מהקוד החי אל `admin-data.js`. אחרי שינוי בבנק השאלות /
במאפיינים / באיים / במשחקונים / בשאלוני הפולס — מריצים מחדש:

```
node admin/build-admin-data.js
```

ואז מרעננים את הדף.

## מה יש בפנים

### 🌊 אבני יסוד
- **בנק שאלות** — 864 שאלות, היררכיה: תחום (strand) → נושא → תת-נושא → שאלות, עם סינון לפי רמה/חיפוש.
- **האיים** — לפי הסדר, כרטיס לכל אי: מה נבדק, כמה משחקונים (בקוד + מתוכננים), אילו מכניקות, וקישורים.
- **משחקונים** — טבלה של כל ה-`stage-*.html`: מכניקה, **קול** (נוני מוקלט / הקראת-דפדפן⚠),
  **מנוע** (BKT/EPA — האם מזין את המנוע ולאיזו מיומנות), קישור לפתיחה.
- **טבלת QA** — צ'ק אודיו/תמונות/טעינה + **קול** (איזה קול יש בכל משחקון) לכל משחקון.
  - ברירת מחדל = בדיקה אוטומטית (נכסים מקומיים חסרים; זיהוי-קול לפי audio.js/mp3 מול speechSynthesis).
  - הסימון הידני (✓) נשמר ב-`localStorage` (`admin-qa-overrides-v1`) וגובר על האוטומטי.
  - 🔊 "הקראת דפדפן / מעורב ⚠" = משתמש ב-Web Speech (עברית גרועה/שקט) — חובה לבדוק.

### 💜 פולס
- **כלי הפולס** — כל השאלונים/דשבורדים + קישורים.
- **שאלות** — אותם 4 ממדים, מנוסחים בנפרד לתלמיד.ה / מורה / הורה.
- **תשובות תלמידים** — נקראות מ-`localStorage` במפתח `pulse-demo-responses` — אותו מפתח
  שהשאלון החי (`pulse-grade1/pulse-questionnaire.html`) כותב אליו. כשתלמיד.ה משלים.ה שאלון,
  התשובות מצטברות ומוצגות כאן (צבירה פר ממד + טבלה גולמית).

## מבנה
- `index.html` — כל ה-UI (טאבים + רינדור). ניווט עם hash: `#avnei/av-islands`, `#pulse/pl-responses` וכו'.
- `admin-data.js` — פלט אוטומטי, **לא לערוך ידנית**.
- `build-admin-data.js` — הסקריפט שמחלץ את הכל.
